# Dokumentation – IoT Plant Admin

Dieses Verzeichnis enthält die vollständige Projektdokumentation für die Abgabe.

## Empfohlene Lesereihenfolge

1. [PROJEKTDOKUMENTATION.md](PROJEKTDOKUMENTATION.md)  
  Vollständige Hauptdokumentation (Zielumfang 10–12 Seiten) mit Architektur, Frontend/Backend-Funktionsweise, Datenfluss, Deployment und Übergabe.

3. [API.md](API.md)

4. [ARCHITECTURE.md](ARCHITECTURE.md)

5. [HARDWARE.md](HARDWARE.md)

6. [HANDOVER.md](HANDOVER.md)

Die Dokumentation soll ermöglichen, dass neue Studierende das Projekt ohne mündliche Übergabe verstehen, lokal starten und fachlich/technisch weiterentwickeln können.
