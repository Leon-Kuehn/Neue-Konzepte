# Projektdokumentation

Dieser Ordner sammelt alle ausfuehrlichen Dokumente fuer die Gesamtloesung.

## Empfohlene Unterstruktur

- `architecture/` Architektur, Komponenten, Datenfluesse
- `integration/` Integrationsleitfaeden fuer Repos und APIs
- `operations/` Deployment, Docker, Betrieb, Monitoring
- `feature-specs/` Fachliche Spezifikationen je Feature
- `decisions/` Architekturentscheidungen (ADR)

## Empfohlene Startdokumente

1. Systemuebersicht
2. Repositories und Verantwortungen
3. Build-/Run-Anleitung
4. API- und Topic-Konventionen (MQTT)
5. Teststrategie
6. Release-Checkliste
